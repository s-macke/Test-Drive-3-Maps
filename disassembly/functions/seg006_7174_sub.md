; sub_14FC4 resolves a render/geometry selector into one of three descriptor banks.
;
; Input split:
; - if AH != 0, AX selects the object bank at word_2D83C/2D83E
; - if AH == 0 and AL < 0x40, AL selects the tile bank at word_2DAD0/2DAD2
; - if AH == 0 and AL >= 0x40, (AL - 0x40) selects the second tile bank at word_2DA60/2DA62
;
; After resolving the pointer, the routine reads bytes +0/+1 as polygon/point counts and then
; executes add si, 3. Since SI was already incremented once after reading byte +0, this lands
; on start+4, so this path parses the common 4-byte header form before calling sub_1528D/sub_15153.
;
; The tile-bank paths also accept a small word value (<= 0x10) in the pointer table and save it
; to word_2AD0E before falling back to the previous entry. The later polygon expander sub_15153
; subtracts this value from the polygon count, so it behaves like tile-only auxiliary metadata.
;
; This is the closest traced runtime equivalent to the viewer's tile-vs-object split, but it is
; not an exact match for the hardcoded per-entry isobj list in src/shared/objects.ts.
