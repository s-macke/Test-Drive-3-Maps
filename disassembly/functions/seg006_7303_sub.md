; sub_15153 appends expanded polygon records into the 0x0A-byte runtime buffer.
;
; Per source polygon (8 bytes):
; - +0: word0 = type bits + flags + index0, copied with the vertex-base offset added
; - +2: word1 = color0 in bits 15:11 + index1, high bits stripped after extracting color0
; - +4: word2 = color1 in bits 15:11 + index2, high bits stripped after extracting color1
; - +6: word3 = material/effect in bits 15:11 + index3, copied intact
;
; The routine then maps color0/color1 through the 32-entry table at word_2D8A0
; and stores the derived fill word at runtime record +8.
;
; This confirms the raw file stores 8-byte polygon records that expand to 10-byte
; runtime records (4 source words + 1 derived fill word).
